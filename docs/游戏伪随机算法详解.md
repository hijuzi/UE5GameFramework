# 游戏开发中的伪随机算法详解

> 核心哲学：**真正的随机 = 完全不可预测 = 糟糕的玩家体验。**
> 伪随机算法的目标不是"看起来像随机"，而是"让玩家感觉公平且有掌控感"。

## 1. PRD - 递增概率算法

### 通俗理解

你玩一个"20% 暴击率"的游戏。真随机可能让你砍十刀不暴击，也可能刀刀暴击。**PRD 的做法**：故意把初始概率降到很低（如 5.57%），每次不暴就往上加一点。你不会憋太久，也不会连暴。

> **一句话**：越倒霉系统越帮你，刚走运就让你缓一缓。

### 数学原理

```
P(N次失败后触发) = C × N     （C 为初始基础概率）
标称 20% → C ≈ 5.57%
```

| 攻击次数 | 本次暴击率 | 累积触发率 |
|:--------:|:----------:|:----------:|
| 第 1 刀  | 5.57% | 5.57% |
| 第 3 刀  | 16.71% | 30.1% |
| 第 5 刀  | 27.85% | 60.4% |
| 第 8 刀  | 44.56% | 91.1% |
| 第 10 刀 | 55.70% | 98.0% |

![PRD 递增概率曲线](./prd_curve.png)

### 游戏运用举例

**Dota 2 / LoL - 暴击机制**

虚空假面出水晶剑（20% 暴击）：

```
真随机体验：
  第一波：× × × × × × × × ✓ ×   （9刀1暴，"这装备假的吧？"）
  第二波：✓ ✓ × ✓ ✓ × × × × ×   （4刀3暴，对手："开挂？"）
  → 体验过山车，双方都不爽

PRD 体验：
  第一波：× × × × ✓ × × × × ✓   （节奏稳定）
  第二波：× × × × ✓ × × × × × ✓ （约5刀一个暴击）
  → 双方都有预期，博弈感更强
```

**其他应用**：PA 模糊（闪避）、斧王反击螺旋、雷锤触发等所有"概率被动"。

## 2. 保底机制 - Pity Timer

### 通俗理解

你妈说："这次考不好，下次帮你补习。"你越考不好，补习力度越大。到第 10 次直接帮你答题——这就是保底。

- **软保底**：概率慢慢往上加
- **硬保底**：第 N 次 100% 必中

> **一句话**：给玩家的运气设置一个"最差下限"。

### 游戏运用举例

**原神 / 星穹铁道 - 抽卡系统**

```
基础五星概率：0.6%（纸面公示）
  第 1-73 抽：每抽 0.6%
  第 74 抽：约 6.6%（软保底开启）
  第 75 抽：约 12.6%
  第 76 抽：约 18.6%
  ...
  第 90 抽：100%（硬保底）

实际平均：62.5 抽出五星，综合概率约 1.6%
```

**为什么有效？**
- 纯随机 0.6% → 约 5% 的玩家会 300 抽不出 → 直接弃坑
- 保底让最非的人也有确定性希望 ——"再攒 10 抽就保底了"
- 花钱冲动来自"差一点就保底"的确定性，而非"也许能中"的虚无

**王者荣耀 - 荣耀水晶（361 保底）**：同样的逻辑。

**Diablo 4 - 稀有掉落保底**：BOSS 打 100 次没掉 → 掉率提升 → 200 次继续提升 → 保证不会"刷一辈子不出"。

## 3. Shuffle Bag - 洗牌袋算法

### 通俗理解

把一副扑克洗好，一张张翻开，翻完再洗。每轮每张牌恰好各一次，不重不漏。

> **一句话**：先定好这轮有哪些东西，乱序排列按顺序给，给完再定下一轮。

### 游戏运用举例

**俄罗斯方块 - 7-Bag 系统（最经典应用）**

```
袋子：[I, O, T, S, Z, J, L]（7 种方块各一个）
洗牌 → [T, L, I, S, Z, O, J]
玩家依次拿到：T → L → I → S → Z → O → J
7 块拿完，重新装袋洗牌。

如果纯随机：I I S S S S Z Z Z Z
→ 玩家连出 4 个 S、4 个 Z，场上洞越来越大，直接 GG
→ "这游戏针对我！"

7-Bag 保证：
  ✅ 绝对不会连续出现同一种方块
  ✅ 每 7 块内种类均衡
  ✅ 玩家可以规划策略
```

**音乐播放器 - 随机播放**

```
用户期望的"随机"：每首歌都会轮到，不重不漏
  真随机：A B B A A C C A B（B 反复出现，D 永远听不到）
  → 用户投诉："随机有 Bug！一直循环那几首！"

Spotify 实际做法：整个播放列表做成 Shuffle Bag
  → 一轮听完所有歌前不重复
  → 用户感知："这才是真正的随机！"
```

**卡牌游戏抽牌**：炉石 30 张牌组洗好一张张抽，本质就是 Shuffle Bag。

![真随机 vs Shuffle Bag 分布对比](./shuffle_bag.png)

## 4. 不放回抽样

### 通俗理解

袋子里 5 红 3 蓝 2 绿，摸一个拿出来**不放回去**。已抽出的暂时移除，保证短期内不重复。

> **一句话**：抽过的东西暂时拿掉，给所有东西轮流登场的机会。

### 与 Shuffle Bag 的区别

| | Shuffle Bag | 不放回抽样 |
|---|---|---|
| 每轮结构 | 严格每种各一个 | 每轮数量自由 |
| 确定性 | 高（全种类必出） | 中（未必全种类） |

### 游戏运用举例

**NPC 语音/台词系统**

```cpp
池子 = ["吃我一刀!", "看招!", "受死吧!", "别跑!", "哈!"]

第 1 次攻击 → 抽 "受死吧!"（移除）
第 2 次攻击 → 抽 "哈!"（移除）
...
第 5 次攻击 → 池空了，重新填充

→ 5 次攻击 5 句不同台词，NPC "会换着花样说"
→ 如果用真随机：3 秒内连续三次"吃我一刀！"→ NPC 像个复读机
```

**Boss AI 行为**

```
Boss 有 3 招：普攻、重击、冲锋
不放回抽样：重击 → 普攻 → 冲锋（轮流用）
→ 不会连续用同一招
→ 玩家有时间适应每种攻击
```

**暗黑破坏神 - 事件刷新**：今天遇到哥布林祭坛 → 下次遇到概率降低 → 提高遇到盗宝地精等没遇过的事件概率。

## 5. 滑动窗口约束

### 通俗理解

你是厨师，要求"最近 3 天必须有鱼有肉有菜，每样至少一次"。安排明天菜单时，看过去两天吃了啥——还差哪样没吃，就做那样。

> **一句话**：盯住最近 N 次历史，快漏掉的就赶紧补上。

### 游戏运用举例

**Roguelike - 房间类型生成**

《杀戮尖塔》、《黑帝斯》：

```
窗口 = 最近 7 个房间
规则：必须包含 ≥1 商店、≥1 休息点、≥1 精英、≥1 事件

第 8 个房间生成时：
  检查最近 7 个 → 没有商店！
  → 第 8 个必须是商店
```

效果：避免"全程没商店攥着钱用不出去"或"全是精英根本打不过"。

**MOBA - 小兵路线**

LOL 中最近 5 波小兵左右偏差 ≤ 2，保证兵线不会一直偏一边。

![滑动窗口约束示意图](./moving_window.png)

## 6. 赌徒谬误修正

### 通俗理解

你最近特别倒霉，系统主动提高你的暴击率，让你"回归平均"。反过来运气爆棚就稍微压一压。

> **一句话**：统计你最近的运气，往"回归均值"方向微调。

### 游戏运用举例

**MMORPG - 装备强化**

```
+7 上 +8，理论 30%：
  最近 10 次全失败（理论 3 次成功，实际 0 次）
  → 修正 +10% → 第 11 次概率 = 40%
  → "还好我没放弃"

  最近 4 次成了 3 次（太欧了）
  → 修正 -8% → 下次概率 = 22%
```

**魔兽世界 - 伤害浮动**

```
基础 100，浮动 ±10
第 1 刀 108（偏高）→ 系统记录 +8 → 第 2 刀倾向偏低
第 2 刀 95 → 偏差 +3 → 第 3 刀 98
最终伤害在 100 附近小幅波动，不会大起大落
```

![赌徒谬误修正概率调节](./gambler_correction.png)

## 7. 双阈值限制

### 通俗理解

给随机加"紧箍咒"：两次触发之间至少隔 X 次（冷却），最多 Y 次必触发（保底）。

> **一句话**：不能太快，不能太慢，必须在"黄金窗口期"内发生。

### 游戏运用举例

**MOBA - 被动技能冷却**

```
斧王反击螺旋（17% 触发）：
  冷却：连续触发至少间隔 1 次攻击
  保底：8 次没触发 → 下一次直接触发

斧王跳进 5 人堆：
  → 触发！啪！
  → 0.3 秒后...（冷却中，不触发）
  → 0.5 秒后...触发！啪！
  → 不会 0.1 秒连爆 3 次，也不会 3 秒不爆
```

**FPS - 散弹枪散布**

```
《Apex》散弹枪：
  下限：弹丸之间不能贴太近
    → 保证近距离不是"所有弹丸打一个点"（太 OP）
  上限：弹丸不能飞出准星范围
    → 中距离也至少命中 X 颗
    → 每把枪的手感稳定可预期
```

## 8. 蓝噪声与泊松碟采样

### 通俗理解

在草坪上撒豆子：
- **白噪声（真随机）**：这里一堆，那里一片空
- **蓝噪声**：豆子间距都差不多，均匀撒满，但不是排成方阵
- **泊松碟采样**：蓝噪声的一种实现，明确指定"任意两点间距 ≥ 最小半径"

> **一句话**：让东西均匀撒开，避免扎堆和空白。

### 游戏运用举例

**开放世界 - 植被/资源点分布**

```
白噪声放苹果树：
  区域 A：8 棵扎堆（溢出）| 区域 B：0 棵（荒芜）
  → 体验两极分化

蓝噪声放苹果树：
  每棵间隔 50-80 米，均匀分布
  → 走哪都能看到一棵，探索体验流畅
```

**《艾尔登法环》- 怪物营地分布**

```
泊松碟采样（最小距离 200 米）：
  🏕️    🏕️    🏕️    🏕️
      🏕️    🏕️    🏕️
  🏕️    🏕️    🏕️    🏕️

→ 走几步能遇到战斗，但不会连锁拉两营怪
→ 每个营地之间有空余供休整
```

**TAA 抗锯齿 - 蓝噪声抖动**

每帧采样点做蓝噪声偏移，多帧合并等效超采样。如果用白噪声抖动 → 画面会闪烁，蓝噪声 → 稳定清晰。

## 9. 加权随机

### 通俗理解

不是所有结果等概率，给每种结果分配"话语权"（权重）。权重大的多出，权重小的真稀有。

> **一句话**：重要的事让它多发生，稀有的让它真稀有。

### 游戏运用举例

**掉落表（Loot Table）**

```
普通哥布林掉落表：
┌──────────┬────────┬──────┐
│   物品   │  权重  │ 概率 │
├──────────┼────────┼──────┤
│ 金币 ×10 │  500   │ 50%  │
│ 金币 ×50 │  300   │ 30%  │
│ 白色装备 │  150   │ 15%  │
│ 蓝色装备 │   45   │4.5%  │
│ 紫色装备 │    4   │0.4%  │
│ 传说装备 │    1   │0.1%  │
└──────────┴────────┴──────┘
→ 大部分时候掉金币，偶尔惊喜
```

**动态权重进阶**：

```
1 级玩家打哥布林：
  紫色装备权重 = 0（级别不够，不会掉落）
  → 避免新手第一只怪出神装

100 级玩家打哥布林：
  白色权重 450 → 100（低级的少掉）
  蓝色权重 15 → 60（匹配当前等级）
  → 同一只怪，不同阶段掉不同东西
```

**Boss AI 行为选择**

```
中距离时：
  火球 40% | 召唤小怪 25% | 突进 20% | 大招 10% | 嘲讽 5%

玩家贴脸后动态调整：
  火球 → 100（贴脸放远程不合理，降权）
  近战横扫 → 500（新增，近战主力）
  → Boss "看起来有脑子"
```

## 10. 分层采样

### 通俗理解

先把结果分大类（层），先决定从哪层抽，再在层内抽具体的。就像吃饭：先决定吃面还是吃饭，再决定牛肉面还是炸酱面。

> **一句话**：先确保大方向不出偏差，再在大方向内细化。

### 游戏运用举例

**抽卡系统**

```
十连抽过程：
第 1 步：先定每抽稀有度（按稀有度总权重）
  → R, R, SR, R, R, R, SR, R, R, R = 8R + 2SR ✓

第 2 步：每层内随机选具体角色
  R 池：安柏/凯亚/丽莎/芭芭拉/诺艾尔（各 20%）
  SR 池：班尼特/行秋/香菱

如果纯加权混抽：可能 9R + 1SSR
  → 虽然 SSR 更值钱，但"说好的保底 SR 呢？"
  → 分层保证大框架不崩
```

**Roguelike 地图生成**

```
第 1 层：决定结构
  1 起点 + 1 BOSS + 5 战斗 + 2 事件 + 1 商店

第 2 层：每类内部随机
  BOSS：哥布林王/骷髅将军/史莱姆王（三选一）
  战斗 ×5：从 20 个模板随机
  事件 ×2：从 15 个模板随机

→ 每局都有完整探索节奏
```

## 11. 种子随机

### 通俗理解

"种子"就是一本书的页码。同样页码翻到同样的随机数序列。给随机"拍照存档"，相同种子 → 完全一样的结果。

> **一句话**：可复现的随机——同样的输入永远得到同样的输出。

### 游戏运用举例

**Minecraft - 世界生成**

```
种子 -8376284617392017362
→ 相同地形、生物群系、村庄、要塞位置
→ 朋友输入同种子 = 完全一样的世界
→ 社区分享"神种子"，Speedrun 练习最优路线
```

**Roguelike - 每日挑战**

```
日期 2026-06-30 → 哈希 → 种子 38472910
决定：房间布局、怪物、宝物掉落、BOSS

所有玩家同一天打同一个种子：
  → 公平竞技（比决策而非运气）
  → 排行榜有意义（相同条件最优解）
```

**网络同步 - 弹道散布**

```
服务器和客户端用相同种子：
  只要同步"第 N 发子弹 + 时间 T"
  → 客户端自己算出相同偏移量
  → 不需每发子弹传坐标（大幅省带宽）
```

**星际争霸 2 - 回放系统**：回放文件只有几十 KB，只存操作指令 + 种子，重演一遍得到完全一致的结果。

## 12. 噪声函数

### 通俗理解

- **Perlin / Simplex Noise**：相邻位置的值平滑过渡，产生连绵起伏的效果
- **Voronoi Noise**：产生"细胞分裂"般的纹理

> **一句话**：用数学函数生成"看起来随机但连续自然"的值。

### 游戏运用举例

**Minecraft - 地形生成**

```cpp
height = PerlinNoise(x, z) * 64  // 产生连绵山脉
树的密度 = SimplexNoise(x, z)     // 有的地方密，有的地方疏
```

多层噪声叠加（Octaves）产生不同尺度的细节：大尺度 = 山脉位置，中尺度 = 丘陵，小尺度 = 地面起伏。

**材质/纹理**

```
大理石纹理 = PerlinNoise(x, y)
木头年轮 = VoronoiNoise(x, y)
火焰效果 = 多层 Perlin 叠加 + 垂直偏移
```

**程序化动画**

```
草的摆动 = sin(时间 + PerlinNoise(x, z))
  → 相邻草摆动幅度接近但不同
  → 风吹草动的自然感
```

## 13. 算法速查对照表

| 需求场景 | 推荐算法 | 解决什么问题 |
|----------|----------|-------------|
| 暴击/闪避/被动触发 | PRD + 保底 | 消除连爆/连空极端 |
| 抽卡 SSR | 分层保底（软+硬） | 最非也有确定性希望 |
| 俄罗斯方块 | Shuffle Bag (7-bag) | 永远不会卡方块 |
| 音乐随机播放 | Shuffle Bag | 听完一轮不重复 |
| NPC 台词/音效 | 不放回抽样 | 不连续重复同一句 |
| Boss AI 行为 | 不放回 + 冷却 | 招式有节奏不单调 |
| Roguelike 房间 | 滑动窗口 | 探索节奏完整 |

## 核心设计原则

> **玩家不在乎概率是多少，玩家在乎"这公平吗"。**

1. **消除极端体验** —— 不要让玩家连续 20 次不暴击
2. **可预测的节奏感** —— 让随机有"呼吸感"
3. **短期可控，长期准确** —— 短期分布均匀，长期期望正确
4. **透明度** —— 让玩家感知到机制存在（如"保底计数器"）
5. **确定性希望** —— 让玩家知道"再坚持 X 次就一定能成"
